There has been some debate within cancergrid work package 1 about administered item administration records.

Literal interpretation of the standard results in two administered_item_administration_records per class: one inherited from the administered_item class and another named specifically for the class itself (such as classification_scheme_administration_record).  We can see no need for this, and have chosen to omit the directly named administration record and inherit the administration records for modelling simplicity and to retain the greatest correspondence between the representation of the ISO11179 model and the published standard.

If you have comments on this issue, I would be grateful to discuss.  Please send them to steve.no.spam.harris@comlab.ox.ac.uk ommitting no.spam
