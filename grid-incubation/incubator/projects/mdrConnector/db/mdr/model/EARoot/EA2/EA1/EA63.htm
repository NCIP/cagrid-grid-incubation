Association directions have been added to the model to prevent cyclical references in generation of the ISO11179 schema

Association end lables have been inverted from the UML digrams in the standard to obtain the correct naming of schema elements and attributes.  In this diagram, this creates the additional problem on schema generation that language_section would have two 'containing' attributes.  These have been renamed to avoid this problem.

Association classes are not properly handled by EA schema generation so these are expanded here
