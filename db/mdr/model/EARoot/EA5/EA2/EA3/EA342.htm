As an administered item, the context class inherits the normal terminological entry structure, and thus can have several names and definitions in different languages.

So why does it also have a 'Context Description' attribute?  And the language identifiers for the Context Description?  Surely the Context Description is the preferred description of the administered item?  

Consequentially, we have removed these attributes in our implementation.
