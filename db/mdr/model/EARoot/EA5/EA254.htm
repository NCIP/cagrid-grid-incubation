This diagram is the analog of the 'consolidated metamodel' in ISO 11179.  It displays all of the functional classess that define a data element and its value domain.  The major change from the full ISO11179 model is that relationships between conceptual domains are assumed to be encapsulated in the vocabulary/thesaurus/ontology serving the resource with concepts.  Thus the optional conceptual domain -> conceptual domain association class is omitted.<br /><br />In this diagram the associations broken into key-reference pairs are expressed as dependencies given the relationship name of the foreign key.  Where the relationship is non-obvious, the target class has been added to the diagram.
